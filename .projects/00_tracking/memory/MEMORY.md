- **Pulse architectural placement (S21):** top-level `src-tauri/src/cross_section/` Rust module, NOT under `analysis/`. Distinct sidebar slot + likely growth of percentile-related compute justifies separation. `analysis_tools` registry row (`pulse`, display_order 0) included for consistency — costs one seed line, gains uniform `enabled` toggle infrastructure. - **Universal indicator compute (S21):** Pulse calls `find_indicator(id).compute(&bars, &json!({...defaults}))` directly — bypasses per-ticker `indicator_settings.enabled` flags AND per-ticker params overrides. Reasoning: cross-section reads break if half the rows use different SMMA confirm_bars or RSI lengths. Different from feature-chart compute (which respects per-ticker overrides). Precedent for any future cross-asset enumeration view.
- **Two distinct color axes principle (S21):** REGIME = categorical state via SMMA palette tokens (user-customizable per S9). Heatmap cells = continuous percentile via fixed Palette 1 (R→Y→G). They serve different semantic roles, don't have to match. Repainting chips to R/Y/G would silently override user palette customization — explicitly wrong. Mitigation: beefier chip presence (min-width 84px, alpha 0.28 bg, border alpha 0.7) so it self-anchors.
- **Saturation curve = neutral-hold (S21):** 40-60 stays muted (alpha ramps 0 → 0.06), then ramps to 0.55 at the tails. `pulseCellBg(percentile)` in `PulseDashboard.tsx`. Linear-from-50 alternative considered + rejected — too uniform, dilutes "spot the outlier" use case.
- **"Your universe" not "your watchlist" (S21):** WATCHLIST is a sector_group (the personal-additions slot, empty by default). Pulse iterates EVERY ticker × EVERY sector_group + EVERY FRED series. "Universe" doesn't collide with the sidebar label.
- **Coverage thresholds (S21):** `<30 bars` → no_bars greyed row. `<252 bars` (1y) → partial_history asterisk. Macro `<60 obs` → partial. Above 252 → full percentile compute.
- **Ticker-column click only (S21):** picked over whole-row click. Per-cell hover tooltips are part of Pulse's value prop (raw value + percentile + lookback + last-fetched per design doc), so whole-row click would put every cell hover one accidental misclick from navigation. Apps that do whole-row click (Gmail, Linear, Slack) don't put per-cell tooltips inside the row — convention doesn't transfer. Bloomberg HRH / TradingView heatmaps treat ticker as hyperlink, cells as read-only. CSS: `.pulse__ticker--clickable` gets cyan-on-hover + underline + cursor-pointer. Macro + no-bars rows render the ticker as a non-clickable `<div>`.
- **Pulse → TickerDashboard handoff (S21):** localStorage key `session.pulse_feature_chart_target = JSON.stringify({ ticker, dataSource })`. Same shape as S17 Correlations→Pairs handoff. TickerDashboard reads + clears the handoff after tiles load — `setSelected(matchingTile)` opens the inline feature chart. Always-clear-on-mount (success or mismatch) prevents stale handoffs hijacking later manual navigation. Required adding `data_source` field to `CrossSectionRow` so the handoff key is exact (same ticker can exist under different sources).
- **Stale-section fallback (S21):** App.tsx watches `(groups, activeSection)` and redirects to `'pulse'` when `activeSection` doesn't resolve in groups. Covers legacy `'scanner'` and any user-deleted custom group.
- **Scanner deprecation (S21):** Pulse subsumes Scanner's analytical content. Scanner's unique value reduced to PRIME (critical, moved to Pulse banner contextually), RECOMPUTE (dead weight — killed), raw RSI/ATR/price (debatable — percentile lens more useful for cross-asset). Scanner unrouted from sidebar + App.tsx; soft-deleted in seed via `UPDATE sector_groups SET user_hidden = 1 WHERE id = 'scanner'`. Scanner.tsx + IndicatorScanner + scanner_snapshot IPC preserved one release for revert-safety; delete in v1.3 cleanup. Same rationale that killed feature #5 (Watchlist Performance) in S10.
- **PRIME-in-Pulse contextual UX (S21):** amber chip in filter row, only renders when `noBarsCount > 0`. After PRIME completes, `result.failures` rendered as a list under the prime-status strip — one row per failure with monospace ticker + error + exchange-suffix hint ("Symbols that consistently fail are likely listed on a different exchange than the seeded suffix indicates"). ### v1.2 UX leverage week (S22)
- **Watermark `'center'` keyword vs numeric `'50%'` (S22, see LESSONS EC-17):** ECharts graphic text element with `left: '50%'` anchors the bounding box's left edge, NOT its center, even with `textAlign: 'center'`. The keyword `'center'` triggers ECharts auto-centering. Line-mode watermark used `'center'` and worked; candlestick path used numeric `'50%'` after S19's vertical-pin work. Fix: use keyword for horizontal, keep numeric `topPct` for grid-pinned vertical (`textVerticalAlign: 'middle'` does work with numeric percentages — asymmetric ECharts behavior).

Built cross-link instead — `tile_visible = 1` flip + click navigates to ANALYSIS tab via `session.analysis_handoff_tab` localStorage handoff. Shared component is yak-shaving for two series; cross-link delivers user value cheaply. Build the shared component only when a third series wants the dual treatment.
- **Three-handoff localStorage pattern locked (S22):** localStorage > CustomEvent for cross-section navigation because the receiving component unmounts/remounts during the section flip. A synchronous `setActiveId(target)` from a handoff useEffect runs BEFORE the load resolves — load then overwrites the handoff. Fix: gate the handoff useEffect on `activeIdStatus.loaded`. Pattern applies anywhere a `usePersistedState` value is set externally on mount.
- **Anchored VWAP shape (S22):** transient anchor (lost on toggle-off + bar change — exploratory by design); whole-pane click via ZRender (`chart.getZr().on('click')` filtered through `chart.containPixel({gridIndex: 0}, [x,y])`); zero-volume auto-suppress (cumulative volume 0 → all-null result → series push skipped); accent-amber color via new `chartTheme.accentAmber` token. First **interactive** chart element in the app — sets the precedent for future click-based affordances.
- **ZRender click handler closure-over-refs pattern (S22, see LESSONS FE-6):** click handler attached **once** in a useEffect with empty deps; refs (`showAvwapRef`, `barsRef`, `modeRef`) feed the latest values to the single attached handler. Re-attaching on every render races ECharts' internal pointer-event state during drag-pan. Reusable for any future ZRender-level event handler that needs current-state access.
- **ECharts `convertFromPixel` finder shape (S22, see LESSONS EC-16):** `{xAxisIndex: 0}` finder expects single number; `{gridIndex: 0}` finder expects 2D `[x, y]` and returns 2D `[xValue, yValue]`. Mixing them silently returns NaN — destructuring `const [v]` extracts undefined and downstream rounds to 0/NaN. Use `{gridIndex: 0}` for click-on-pane → category-index conversions. Same finder for `containPixel` boundary checks.
- **`quote_cache.last_fetch_error` schema location (S22):** new TEXT column on `quote_cache` (per `(ticker, data_source)`), NOT on `watchlist_tickers` (per `(ticker, sector_group_id)`). Fetch state is per-data-source (same ticker can fail at Yahoo and succeed at CoinGecko); per-group keying would duplicate the same error across rows. New `Db::set_quote_fetch_error` upserts a row if absent (first-ever-fetch-failure for new tickers still gets diagnosed) and deliberately doesn't touch `last_fetched` so freshness check keeps treating row as stale. `upsert_quote` clears the column unconditionally on success; PRIME calls `set_quote_fetch_error(None)` separately because `upsert_price_bars` doesn't touch quote_cache.
- **Ctrl+K palette four-surface aggregator (S22):** searchables = tickers + sectors + FRED series + Analysis tabs. Tickers via NEW `list_palette_tickers` IPC (one row per `(ticker, sector_group_id)`, deliberately denormalized for unambiguous navigation — `list_tickers_with_coverage` does `SELECT DISTINCT` and is the wrong shape). Fuse.js@7.3.0 (12KB gz) at threshold 0.4 handles typos (`btif` → `BITF.TO`). Aggregator memoizes searchables so Fuse rebuild fires only on real data changes.
